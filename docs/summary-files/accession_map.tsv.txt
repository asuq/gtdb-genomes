Notes:
ncbi_accession records the original requested accession.
download_request_accession records the terminal exact value passed to datasets
for that row.
final_accession records the realised versioned accession from the extracted
payload on successful downloads.
The workflow treats the realised accession as authoritative, so versioned request accessions fail closed if the exact realised accession is absent after extraction.
Only versionless request accessions may accept a unique same-family realised version.
Unsupported legacy UBA* rows leave download_method_used and download_batch
blank because the workflow skips execution for them.
